Metodo de Seleccion Directa

Es un algoritmo relativamente sencillo y uno de los más fáciles de recordar e implementar.

Se basa en realizar varias pasadas, intentando encontrar en cada una de ellas el elemento que según el criterio de ordenación es mínimo y colocándolo posteriormente en su sitio.

Sencillo y practico

Este metodo es tan sencillo, que no tiene un autor al que se le haya atribuido su creacion.

Adaptable:

Funciona en cualquier sistema y cualquier tipo de lenguaje.

Una ventaja es la memoria que utiliza, ya que solo requiere reservar memoria para un elemento mas del tipo del arreglo que se este ordenando.

Evolucion de la Burbuja

Este metodo es una mejora al metodo burbuja, la diferencia es minima.

No es el mejor, pero si el mas facil

Por la naturalidad del metodo, puede que a muchas personas se les haya ocurrido utilizarlo sin siquiera conocerlo, es normal pero no por esto quiere decir que el algoritmo sea realmente bueno.

Su tiempo de ejecucion siempre es el mismo en cualquiera de los casos, O(nˆ2).

Ventajas

Facil y poca memoria.

-Efectivo en pocos datos
-Necesita poca memoria
-Ideal en los que intervengan dispositivos externos de almacenamiento.

Desventajas

Demasiadas comparaciones.

-No importa si se manejan pocos datos, el orden sera el mismo.
-Desaprovecha arreglos que tienen tendencia al orden

Origen

Desde el principio de los tiempos

Debido a su naturalidad, su creacion no se le atribuye a ninguna persona en especifico

Pasos del Algoritmo

Pasos descritos

-Buscar el mínimo elemento de la lista
-Intercambiarlo con el primero
-Buscar el siguiente mínimo en el resto de la lista
-Intercambiarlo con el segundo

Codigo en C

Codificacion contando con las condiciones ideales.

void SeleccionEnteros(int[] A)
{
int iaux, posMin n = A.Length;
for (int i = 0; i < n - 1; i++)
{
posMin = i;
for (int j = i+1; j < n; j++)
{
if (A[j] < A[posMin])
{
posMin = j;
}
}
iaux = A[i];
A[i] = A[posMin];
A[posMin] = iaux;
}
}

Paso uno

Recorre el arreglo y busca el minimo elemento de la lista.

Paso dos

Se intercambia con el primer elemento de la lista.

Paso tres

Buscar el siguiente mínimo en el resto de la lista

Paso cuatro

Intercambiarlo con el segundo elemento de la lista.

Paso cinco

Buscar el mínimo elemento entre una posición i y el final de la lista

Paso seis

Intercambiar el mínimo con el elemento de la posición i hasta llegar al orden
